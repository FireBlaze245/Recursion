## **Рекурсия**  
**это метод решения задач, при котором функция вызывает саму себя для обработки более простой версии той же задачи.**
<br>
<br>

## Спуск и подъем по стеку вызовов  
Когда функция вызывает себя, каждое новое состояние сохраняется в стеке вызовов (call stack):
<br>

<img width="628" height="290" alt="i" src="https://github.com/user-attachments/assets/857a984f-e585-4e35-94c3-5305b4509521" />


<br>

*Спуск* (погружение)  
- Каждый рекурсивный вызов создает новый "слой" на стеке  
- Сохраняются локальные переменные и параметры  
- Процесс продолжается до достижения базового случая (условия остановки)  
<br>
  
*Подъем* (возврат)  
- После достижения базового случая начинается обратный ход  
- Каждый слой стека "разворачивается"  
- Возвращаемые значения передаются вверх по цепочке  
<br>

Ниже приведена простая реализация:

``` c
void example(int n) {
    printf("Вход: n = %d\n", n);                    // Спуск
    if (n == 0) {                                   // Базовый случай
        printf("Достигнут базовый случай!\n");
        return;
    }
    example(n - 1);                                 // Рекурсивный вызов
    printf("Выход: n = %d\n", n);                   // Подъем
}
```

***

📊 Визуализация стека для factorial(4)

***

factorial(4)  
├── factorial(3)  
│ ├── factorial(2)  
│ │ ├── factorial(1)  
│ │ │ ├── factorial(0)  
│ │ │ │ └── return 1  
│ │ │ └── return 1 * 1 = 1  
│ │ └── return 2 * 1 = 2  
│ └── return 3 * 2 = 6  
└── return 4 * 6 = 24 

``` c
int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1); 
}
```

## Виды рекурсии  

### Прямая рекурсия  

Это самый простой и распространённый вид рекурсии: функция напрямую вызывает саму себя. Такой подход интуитивно понятен и часто используется для задач, которые естественным образом разбиваются на подзадачи того же типа — например, обход дерева, вычисление факториала или поиск в глубину.  

```c
void direct(int n) {
    if (n <= 0) return;          // Базовый случай: останавливаем рекурсию
    printf("Step: %d\n", n);     // Действие на текущем шаге
    direct(n - 1);               // Прямой рекурсивный вызов
}
```

**Важные моменты:**  
* **Базовый случай обязателен.** Без условия остановки рекурсия станет бесконечной и приведёт к переполнению стека (stack overflow).  
* **Глубина рекурсии** зависит от входных данных: при больших `n` может не хватить памяти стека.  
* В простых случаях прямую рекурсию можно заменить циклом — это часто эффективнее по памяти.  

---

### Косвенная рекурсия (взаимная рекурсия)  

Здесь функция не вызывает себя напрямую, а делает это через цепочку других функций. Самый простой случай — взаимный вызов двух функций: `A` вызывает `B`, а `B` вызывает `A`. Такой паттерн встречается, например, при разборе выражений, моделировании конечных автоматов или проверке чётности/нечётности через взаимные вызовы.  

**Пример на C:**  
```c
void A(int n);
void B(int n);

void A(int n) {
    if (n <= 0) return;
    printf("A: %d\n", n);
    B(n - 1);                    // Вызов B
}

void B(int n) {
    if (n <= 0) return;
    printf("B: %d\n", n);
    A(n - 1);                    // Вызов A — замыкает цикл
}
```

**Особенности:**  
* Нужно заранее объявлять функции (forward declaration), чтобы компилятор знал об их существовании.  
* Базовый случай должен быть в обеих функциях (или хотя бы в одной, но так, чтобы цепочка гарантированно прерывалась).  
* Отладка сложнее, чем у прямой рекурсии, потому что поток выполнения «прыгает» между функциями.  

---

### Хвостовая рекурсия  

Хвостовая рекурсия — это особый случай, когда рекурсивный вызов является **последней операцией** в функции: после него ничего не вычисляется и не возвращается дополнительно. Это важно, потому что многие компиляторы умеют выполнять **оптимизацию хвостовых вызовов** (Tail Call Optimization, TCO): вместо создания нового стекового кадра они переиспользуют текущий. В результате глубина стека не растёт, и функция работает почти как цикл.  

**Классический пример — факториал:**  

```c
// Обычный (не хвостовой) вариант:

int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1);  // Умножение ПОСЛЕ рекурсивного вызова
}
// Здесь после возврата из `factorial(n-1)` нужно ещё умножить на `n`, поэтому это не хвостовой вызов.

// Хвостовой вариант с аккумулятором:
int tail_factorial(int n, int acc) {
    if (n == 0) return acc;
    return tail_factorial(n - 1, n * acc);  // Рекурсивный вызов — последняя операция
}

// Начальный вызов: tail_factorial(5, 1)
```

**Ключевые моменты:**  
* Аккумулятор (acc) хранит промежуточный результат и передаётся дальше.  
* При наличии TCO такая функция может обрабатывать очень большие значения n без переполнения стека.  
* Не все языки и компиляторы поддерживают TCO: в C это зависит от компилятора и уровня оптимизации, в Scheme и некоторых функциональных языках — гарантировано.  
* Хвостовую рекурсию почти всегда можно переписать в виде цикла — это даёт предсказуемую производительность.

## **Рекурсия vs Итерация**  

### Итерация  
— это повторение блока кода определенное количество раз или до выполнения условия. В отличие от рекурсии, которая использует вызовы функций, итерация использует циклы (for, while, do-while).  

Основных способов три:
1. for — когда точно знаешь количество повторений.
```c
for (int i = 0; i < 10; i++) {
    printf("%d ", i); // Пройдет 10 раз (от 0 до 9)
}
```
2. while — когда повторяешь, пока условие истинно (проверка в начале).
```c
int i = 0;
while (i < 10) {
    printf("%d ", i);
    i++; // Важно менять условие, иначе бесконечный цикл
}
```
3. do-while — гарантированно 1 раз, потом проверяет условие (проверка в конце).
```c
int i = 0;
do {
    printf("%d ", i);
    i++;
} while (i < 10);
```

### Сравнение рекурсии vs итерации  
| Характеристика | Рекурсия | Итерация |
|:---------------|:----------|:----------|
| Понятность | ✅ Интуитивно понятна | ❌ Сложнее для понимания |
| Память | ❌ Больше (стек) | ✅ Меньше |
| Скорость | ❌ Медленнее | ✅ Быстрее |
| Глубина | ❌ Ограничена стеком | ✅ Не ограничена |
| Код | ✅ Компактный | ❌ Более объемный |

<br>
Главное правило итерации: в теле цикла должно что-то меняться (счетчик, условие), иначе цикл станет бесконечным.
Управление:
break; — вылететь из цикла досрочно.
continue; — перепрыгнуть на следующую итерацию (пропустить остаток тела).

Главное правило рекурсии: Каждая рекурсивная функция должна иметь условие выхода (базовый случай), которое достигается обязательно.
Без этого правила рекурсия будет бесконечной, что приведет к переполнению стека (Stack Overflow) и аварийному завершению программы.


``` c
// Рекурсия
int factorial_rec(int n) {
    if (n <= 1) return 1;           // Базовый случай
    return n * factorial_rec(n - 1); // Вызов себя
}

// Итерация
int factorial_iter(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {   // Цикл
        result *= i;
    }
    return result;
}
```
### Когда использовать рекурсию:  
- Древовидные структуры (файловая система, HTML DOM)  
- Задачи с разбиением (QuickSort, MergeSort)  
- Обход графов (DFS)  
- Задачи, где итерация слишком сложна  

### Когда использовать итерацию:  
- Простые повторяющиеся операции  
- Большая глубина рекурсии  
- Ограниченная память  
- Критичная производительность

## Примеры рекурсии из реальной жизни.  
Быстрая сортировка: 

``` c
void quick_sort(int arr[], int low, int high) {
    if (low >= high) return;                     // Базовый случай
    int pivot = partition(arr, low, high);       // Разделение
    quick_sort(arr, low, pivot - 1);             // Левая часть
    quick_sort(arr, pivot + 1, high);            // Правая часть
}
```

Так же подходит для бинарного поиска

``` c
int binary_search(int arr[], int left, int right, int target) {

    if (left > right) return -1;
    int mid = left + (right - left) / 2;

    if (arr[mid] == target) return mid;
    if (arr[mid] > target)
        return binary_search(arr, left, mid - 1, target);
    return binary_search(arr, mid + 1, right, target);
}
```
Примеры посложнее
<br>
Обход бинарного дерева (DFS) с деструкцией (Освобождение памяти)<br>
Задача: Есть бинарное дерево поиска. Нужно удалить все узлы и освободить память, не создав утечек (Memory Leak).<br>
Сложность: Нельзя удалить родителя, пока не удалены дети (иначе потеряем указатели на них).
```c
#include <stdlib.h>
#include <stdio.h>
typedef struct Node {
    int data;
    struct Node* left;
    struct Node* right;
} Node;

void free_tree(Node** root_ptr) {
    if (root_ptr == NULL || *root_ptr == NULL) {
        return; // Базовый случай
    }

    Node* current = *root_ptr;
    free_tree(&(current->left));
    free_tree(&(current->right));
    current->left = NULL;
    current->right = NULL;
    free(current);
    *root_ptr = NULL; 
}
```
Рекурсивный спуск (Парсинг JSON-подобной грамматики)
<br>
Задача: Дан упрощенный язык арифметических выражений: "(2+(3*4))". Нужно проверить, сбалансированы ли скобки, но с вложенными уровнями сложности (пропуск строковых литералов).
<br>
Сложность: Обычный подсчет ( и ) не работает, если есть экранированные кавычки. Нужен конечный автомат + рекурсия.
```c
#include <stdbool.h>
// Рекурсивная функция проверки вложенности
// Возвращает индекс, на котором остановилась, или -1 при ошибке
int parse_expression(const char* str, int index, int depth) {
    // Базовый случай: глубина вернулась к 0 
    if (str[index] == '\0' && depth == 0) return index;
    // Если закрывающая скобка пришла раньше, чем открывающая - ОШИБКА
    if (str[index] == ')' && depth == 0) return -1;
    if (str[index] == '(') {
        // Углубляемся
        return parse_expression(str, index + 1, depth + 1);
    } 
    else if (str[index] == ')') {
        // Выходим из уровня
        return parse_expression(str, index + 1, depth - 1);
    }
    else {
        // Пропускаем обычные символы
        return parse_expression(str, index + 1, depth);
    }
}
```
Но в реальности парсеры пишут сложнее: они возвращают Абстрактное Синтаксическое Дерево (AST).
<br>
Взаимная рекурсия (Mutual Recursion)
<br>
Это когда A() вызывает B(), а B() вызывает A(). В C это требует предварительного объявления (forward declaration).
<br>
Реальная задача: Определение четности/нечетности числа без операторов % и & (используется в теории автоматов).
```c
#include <stdbool.h>
bool is_odd(int n);
// Функция проверки на четность
bool is_even(int n) {
    if (n == 0) return true;  // Базовый случай
    if (n < 0) return is_even(-n); // Защита от отрицательных
    // Шаг рекурсии: число четно, если предыдущее число нечётно
    return is_odd(n - 1); 
}
// Функция проверки на нечетность
bool is_odd(int n) {
    if (n == 0) return false; // Базовый случай
    if (n < 0) return is_odd(-n);
    // Шаг рекурсии: число нечетно, если предыдущее число чётно
    return is_even(n - 1);
}

// Использование:
// is_even(5) -> is_odd(4) -> is_even(3) -> is_odd(2) -> is_even(1) -> is_odd(0) -> false.
// Глубина рекурсии = n. Это медленно, но демонстрирует механику.
```
Где это применяется в реальности: В компиляторах при разборе выражений типа (a + b) * (c - d), где правила грамматики зациклены.

Рекурсия и указатели  
Обход связного списка (рекурсивно)

``` c
struct Node {
    int data;
    struct Node* next;
};
void print_list(struct Node* head) {
    if (head == NULL) return;       //Конец списка
    printf("%d ", head->data);
    print_list(head->next);         //Переход к следующему
}
```

Рекурсивное удаление списка

``` c
void free_list(struct Node* head) {
    if (head == NULL) return;
    free_list(head->next);             // Сначала удаляем хвост
    free(head);                        // Потом текущий элемент
}
```

## 🎯 Заключение <a id="end"></a>  

Рекурсия — мощный инструмент, который:  
- Делает код чище и понятнее для древовидных структур  
- Позволяет элегантно решать задачи "разделяй и властвуй"  
- Требует осторожности (стек, производительность)  

Главное правило: если задача естественно разбивается на подзадачи того же типа — используйте рекурсию. Если нет — итерацию.
